Create src/guardian/infrastructure.py with functions for idempotent role and channel creation.

Implement ensure_role_exists(guild, role_name):
- Use discord.utils.get(guild.roles, name=role_name) to check existence
- If not found, create with guild.create_role(name=role_name, reason="Guardian initialization")
- Return the role object (existing or newly created)

Implement ensure_channel_exists(guild, channel_name, overwrites=None):
- Use discord.utils.get(guild.channels, name=channel_name) to check existence
- If not found, create with guild.create_text_channel(channel_name, overwrites=overwrites, reason="Guardian initialization")
- Return the channel object

Implement initialize_infrastructure(guild):
- Create @Unverified role (if not exists)
- Create @Verified role (if not exists)
- Create #verify channel with permission overwrites:
  - guild.default_role (@everyone): view_channel=False (deny all)
  - unverified_role: view_channel=True (allow Unverified only)
  - verified_role: view_channel=False (deny Verified, they shouldn't see it)
- Create #security-logs channel (no special permissions)
- Return dict with channel references: {"verify": channel, "security_logs": channel}

Use PermissionOverwrite objects for channel permissions, following RESEARCH.md Pattern 2 example.

Add logging with logging.getLogger(__name__) for infrastructure events (created/found).
