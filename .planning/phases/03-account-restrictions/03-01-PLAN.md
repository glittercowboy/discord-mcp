Constants:
```python
from pathlib import Path

CONFIG_FILE = Path("config/guardian.json")
DEFAULT_CONFIG = {
    "threshold_days": 7,
    "features": {
        "urls": True,
        "attachments": True,
        "role_mentions": True
    },
    "exempt_roles": []
}
```

Functions to implement:

1. `load_config(guild_id: int) -> dict`
   - Check if CONFIG_FILE exists, return DEFAULT_CONFIG.copy() if not
   - Try to load JSON from file
   - Return config for `str(guild_id)` key, or DEFAULT_CONFIG.copy() if key missing
   - Catch `json.JSONDecodeError` and log error, return DEFAULT_CONFIG.copy() on failure
   - Re-reads file on each call for hot-reload support

2. `save_config(guild_id: int, config: dict) -> None`
   - Create parent directory: `CONFIG_FILE.parent.mkdir(parents=True, exist_ok=True)`
   - Use atomic write pattern:
     1. Create temp file: `temp_file = CONFIG_FILE.with_suffix(".tmp")`
     2. Load existing config for all guilds from CONFIG_FILE
     3. Update `all_config[str(guild_id)] = config`
     4. Write to temp file: `json.dump(all_config, f, indent=2)`
     5. Atomic rename: `temp_file.replace(CONFIG_FILE)`
   - Clean up temp file on exception: `temp_file.unlink(missing_ok=True)`
   - Log success/failure with guild_id

Import requirements: `json`, `logging`, `pathlib.Path`

Follow RESEARCH.md patterns:
- Read on each access for hot-reload (no in-memory cache)
- Atomic writes via temp file + rename
- Validate JSON gracefully with try/except
- Store guild_id as string in JSON (JSON keys must be strings)
