<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete emoji-based verification gate system:
- VerificationView with 3 emoji buttons (pizza, taco, burger)
- Moderator bypass logic (administrator/moderate_members/manage_guild)
- Background timeout task (30-second checks, 10-minute deadline)
- Security logging for joins, leaves, verifications, timeouts

The system should:
1. Assign @Unverified to new non-moderator members
2. Send verification challenge to #verify channel
3. Grant @Verified and remove @Unverified on correct emoji
4. Allow retry on wrong emoji (ephemeral error)
5. Auto-kick after 10 minutes if unverified
6. Log all events to #security-logs with timestamps
  </what-built>

  <how-to-verify>
**Prerequisites:**
- Guardian bot running and connected to test Discord server
- Bot has permissions: Manage Roles, Kick Members, Manage Channels, Send Messages

**Test 1: Successful Verification Flow**
1. Create new Discord account (or use alt account)
2. Join test server
3. Verify you receive @Unverified role automatically
4. Check you can only see #verify channel (not main channels)
5. Check #security-logs shows "Member Joined" embed with account age
6. View verification challenge message in #verify
7. Click the 🍕 (pizza) emoji button
8. Verify you now have @Verified role and lost @Unverified
9. Verify you can now see main server channels
10. Check #security-logs shows "Verification Attempt - PASSED" embed

**Test 2: Wrong Emoji Handling**
1. Create another new Discord account
2. Join test server, receive @Unverified
3. In #verify, click 🌮 (taco) or 🍔 (burger) button
4. Verify you see ephemeral error: "Wrong emoji! Try again."
5. Verify you still have @Unverified role
6. Click correct 🍕 button
7. Verify verification succeeds
8. Check #security-logs shows both FAILED and PASSED attempts

**Test 3: Moderator Bypass**
1. Assign yourself Administrator permission in test server
2. Leave and rejoin server
3. Verify you do NOT receive @Unverified role
4. Verify you immediately have access to all channels
5. Check bot logs show "Moderator {name} bypassed verification"

**Test 4: Timeout Auto-Kick**
1. Create new Discord account
2. Join test server, receive @Unverified
3. Do NOT click any verification button
4. Wait 10 minutes and 30 seconds (timeout deadline + check interval)
5. Verify you are kicked from server
6. Check #security-logs shows "Timeout Kick" embed with elapsed time

**Test 5: Security Logging Completeness**
Review #security-logs and verify it contains:
- Member join events (with account age)
- Member leave events
- Verification attempts (success and failure)
- Timeout kicks (if applicable)
- All embeds have timestamps
- All embeds use correct color coding (green=success, red=failure, orange=leave)

**Expected Results:**
- All 5 tests pass without errors
- Bot logs show no permission errors or exceptions
- Security logs are complete and readable
  </how-to-verify>
