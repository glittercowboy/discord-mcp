   ```python

   offense_tracker = offense_tracking.OffenseTracker(Path(__file__).parent / "data" / "offenses.json")
   ```

3. Add link scanning logic to on_message event AFTER account restrictions check (line ~304, after existing deletion/DM logic):
   - Extract URLs using existing URL_PATTERN from account_restrictions (import it)
   - If no URLs found, return early
   - For each URL:
     - Resolve to final destination: `final_url = await link_resolver.resolve_url_destination(url)`
     - Extract domain: `domain = link_resolver.extract_domain_from_url(final_url)`
     - Check blocklist: `if blocklist.is_blocklisted(domain):`
   - If blocklisted domain found:
     - Get current offense count: `offense_count = offense_tracker.get_offense_count(message.author.id)`
     - Delete message (with NotFound exception handling)
     - Record offense: `new_count = offense_tracker.record_offense(message.author.id, message.id, domain)`
     - If new_count == 1 (first offense):
       - Apply 1hr timeout: `await message.author.timeout(timedelta(hours=1), reason=f"Scam link posted: {domain}")`
       - DM user explaining timeout and offense
     - If new_count >= 2 (second+ offense):
       - Auto-ban: `await message.author.ban(reason=f"Scam link offense #{new_count}: {domain}")`
       - DM user explaining ban (will fail after ban, catch Forbidden)
     - Log to #security-logs with embed:
       - Title: "🚨 Scam Link Deleted"
       - Fields: User, Channel, Domain, URL, Offense Count, Action Taken (timeout/ban)
       - Include message content preview (first 100 chars)

4. Use timedelta import from datetime (already imported for account age).

5. Exception handling:
   - Wrap message.delete() in try/except for NotFound (already deleted) and Forbidden (missing perms)
   - Wrap member.timeout() in try/except for Forbidden (missing perms)
   - Wrap member.ban() in try/except for Forbidden (missing perms)
   - Log errors when permission issues occur

Pattern: Follow Phase 3 patterns (silent deletion + DM), Phase 4 patterns (logging to #security-logs with embeds).

Why check links AFTER account restrictions: Account age filter already catches most spam from new accounts. Link scanning is secondary defense for older compromised accounts.

Add slash command to GuardianCommands class for manually reloading blocklist:

```python
@app_commands.command(name="reload-blocklist", description="Reload phishing link blocklist from disk")
@app_commands.default_permissions(administrator=True)
async def reload_blocklist(self, interaction: discord.Interaction):
    """Reload blocklist from file (admin only)."""
    # Import guardian module to access global blocklist instance
    from . import guardian

    # Reload blocklist
    guardian.blocklist.load_blocklist()

    # Count domains loaded
    domain_count = len(guardian.blocklist.domains)

    await interaction.response.send_message(
        f"✅ Blocklist reloaded: {domain_count:,} domains",
        ephemeral=True
    )
```

Pattern: Matches existing slash command patterns (ephemeral response, admin-only permission).

Why manual reload: Blocklist is loaded at startup. If blocklist.json is updated (e.g., downloaded new version), mods can reload without restarting bot.
