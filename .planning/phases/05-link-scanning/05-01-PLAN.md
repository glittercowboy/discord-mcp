  <name>Create link_resolver.py with async redirect following</name>
  <files>src/guardian/link_resolver.py</files>
  <action>
Create src/guardian/link_resolver.py implementing:

1. `async def resolve_url_destination(url: str, timeout_seconds: int = 5, max_redirects: int = 10) -> str`:
   - Follow HTTP redirects using httpx.AsyncClient with follow_redirects=True
   - Try HEAD request first (faster), fallback to GET if HEAD fails
   - Return final URL after redirects (str(response.url))
   - On timeout or error, return original URL (treat as "safe" - don't block on slow servers)
   - Log redirect chain if response.history exists
   - Use timeout=5 seconds to prevent hanging on malicious servers

2. `def extract_domain_from_url(url: str) -> str`:
   - Parse URL with urllib.parse.urlparse to get netloc
   - Extract domain using tldextract.extract(netloc)
   - Return domain.suffix format (e.g., "discord.com" or "bbc.co.uk")
   - Handles complex TLDs correctly (tldextract knows co.uk is a TLD)
   - Return netloc as fallback for IP addresses or unparseable URLs

Pattern: Follow RESEARCH.md Pattern 3 for redirect following, Pattern 2 for domain extraction.

Why httpx over requests: httpx is async-native (no thread pool needed), already in project dependencies. requests is sync-only.
